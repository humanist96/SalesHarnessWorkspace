# SalesHarness — Product Sense

> 이 문서는 제품의 존재 이유와 방향을 정의합니다.
> 모든 설계/구현 결정에서 이 문서의 원칙을 우선합니다.

## 1. Problem Statement (문제 정의)

### 현재 상황 (As-Is)
코스콤 영업직원은 PowerBase 서비스/상품 판매와 증권사 IT 대행 영업을 수행하지만,
체계적인 영업 지원 도구 없이 **엑셀과 한글(HWP)**로 대부분의 업무를 처리하고 있다.

유일한 시스템은 **빌링 관리 시스템**뿐이며, 영업 프로세스 전반을 지원하는 도구는 부재.

### 핵심 페인포인트 4가지

| # | Pain Point | 영향 | 심각도 |
|---|-----------|------|--------|
| 1 | **제안서/보고서 작성에 과도한 시간** | 고객 대면 시간 감소, 영업 기회 놓침 | 🔴 Critical |
| 2 | **고객사 정보 파악 어려움** | 증권사별 IT 현황, 니즈, 히스토리가 산재 | 🔴 Critical |
| 3 | **후속조치 누락** | 미팅 후 팔로우업 지연/누락으로 딜 손실 | 🟡 High |
| 4 | **시장/경쟁 정보 부족** | 최신 트렌드 파악 어려움, 제안 설득력 저하 | 🟡 High |

### 목표 상태 (To-Be)
AI가 반복적인 영업 준비/문서 업무를 자동화하여,
영업직원이 **고객 대면과 관계 구축**에 집중할 수 있는 환경을 만든다.

## 2. Target User (타겟 사용자)

### Primary Persona: 코스콤 영업직원

```
이름: 김영업 (가상 페르소나)
직책: 코스콤 영업팀 과장
경력: 금융IT 영업 7년차
담당: 국내 증권사 5개사 + 해외 증권사 2개사

일과:
  09:00  이메일/메시지 확인, 당일 미팅 준비
  10:00  고객사 미팅 (주 3-4회)
  14:00  제안서/보고서 작성 (하루 2-3시간)
  16:00  내부 회의, 파이프라인 업데이트
  17:00  후속조치 이메일, 다음날 준비

고충:
  - "제안서 하나 쓰는데 반나절이 걸린다"
  - "고객사가 뭘 쓰고 있는지 매번 찾아봐야 한다"
  - "미팅 끝나면 바빠서 후속 이메일을 까먹는다"
  - "경쟁사가 뭘 하는지 알기 어렵다"

기대:
  - "버튼 하나로 제안서 초안이 나오면 좋겠다"
  - "고객사 미팅 전에 브리핑이 자동으로 준비되면"
  - "후속조치를 알아서 알려주면"
```

### Secondary Users (향후)
- **영업팀장**: 팀 파이프라인 현황 조회, 매출 예측
- **경영진**: 영업 성과 대시보드, 전략 의사결정 지원

## 3. Core Value Proposition (핵심 가치)

### One-liner
> "AI가 영업 준비를 하고, 사람은 영업을 한다."

### Value Stack

| 순위 | 가치 | 설명 | 측정 지표 |
|------|------|------|-----------|
| 1 | **시간 절약** | 문서 작성/정보 수집 시간을 80% 줄임 | 제안서 작성 시간: 4시간 → 30분 |
| 2 | **놓치지 않는 영업** | 후속조치 자동 알림으로 딜 손실 방지 | 후속조치 완료율: 60% → 95% |
| 3 | **더 나은 준비** | AI 브리핑으로 미팅 품질 향상 | 고객 만족도, 전환율 |
| 4 | **시장 우위** | 실시간 인텔리전스로 제안 경쟁력 강화 | Win rate 변화 |

## 4. Differentiation (차별화)

### vs. 일반 CRM (Salesforce, HubSpot)
| 구분 | 일반 CRM | SalesHarness |
|------|---------|-------------|
| 데이터 입력 | 수동 입력 중심 | AI가 자동 추출/생성 |
| 문서 생성 | 템플릿 기반 | AI가 맥락 이해하여 자동 작성 |
| 도메인 | 범용 | **금융IT 영업 특화** |
| 인텔리전스 | 기본 리포트 | AI 기반 인사이트/예측 |
| 한국 시장 | 현지화 부족 | **한국 금융시장 맞춤** |

### 핵심 차별화 4가지
1. **금융IT 도메인 특화** — PowerBase 상품 지식, 증권사 IT 용어, 금융 규제 이해
2. **AI 자동화 깊이** — 단순 기록이 아닌, AI가 실제 영업 업무를 수행
3. **코스콤 내부 시스템 연동** — 빌링 시스템 등 사내 데이터와 연결
4. **즉시 사용 가능한 UX** — IT 전문가가 아닌 영업직원도 바로 쓸 수 있는 직관적 인터페이스

## 5. Product Principles (제품 원칙)

> 설계/구현 시 충돌하는 선택지가 있을 때, 이 원칙 순서로 판단한다.

1. **영업직원의 시간을 아껴라** — 클릭 수, 입력 횟수를 최소화. 자동화 가능한 건 자동화.
2. **AI는 보조자이지 결정자가 아니다** — AI가 초안을 만들고, 사람이 검토/수정/승인한다.
3. **데이터는 자산이다** — 모든 영업 활동을 체계적으로 축적. 축적된 데이터가 AI를 더 똑똑하게.
4. **단순함이 이긴다** — 기능을 하나 더 넣는 것보다, 기존 기능을 더 쉽게 쓰게 만들어라.
5. **금융 도메인을 존중하라** — 정확성과 신뢰성이 속도보다 중요. 틀린 정보는 없는 것보다 나쁘다.

## 6. Success Metrics (성공 지표)

### MVP (3개월)
- [ ] 영업직원 5명 이상 주 3회 이상 사용
- [ ] 제안서 작성 시간 50% 단축
- [ ] 사용자 만족도(NPS) 40+

### Growth (6개월)
- [ ] 영업팀 전체 도입 (20명+)
- [ ] 후속조치 완료율 90% 이상
- [ ] 파이프라인 가시성 100% (모든 딜이 시스템에)

### Scale (12개월)
- [ ] 다른 부서/자회사 확산
- [ ] AI 추천 상품의 계약 전환율 측정
- [ ] 매출 예측 정확도 80%+

## 7. What We Don't Do (하지 않는 것)

명확한 경계를 설정하여 범위 확산(scope creep)을 방지한다.

- ❌ **자동 계약 체결** — AI가 계약을 대신 체결하지 않음. 사람이 최종 확인.
- ❌ **범용 CRM** — Salesforce를 대체하려 하지 않음. 코스콤 영업에 집중.
- ❌ **실시간 트레이딩** — 투자 자문이나 트레이딩 기능 없음.
- ❌ **고객 직접 대면 채널** — 이 앱은 영업직원용. 고객이 직접 접근하지 않음.
