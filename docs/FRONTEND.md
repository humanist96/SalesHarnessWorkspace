# SalesHarness — Frontend Guide

> UI/UX 설계 원칙, 페이지 구조, 디자인 시스템, 사용자 흐름을 정의합니다.
> 모든 프론트엔드 구현은 이 문서를 기준으로 합니다.

## 1. Design Direction

### 스타일: 데이터 중심 대시보드

영업직원이 하루에 수십 번 열어보는 도구이므로, **정보 밀도가 높으면서도 읽기 쉬운** 대시보드 스타일.

**참고 레퍼런스:**
- Grafana (데이터 시각화 밀도)
- Linear (깔끔한 네비게이션)
- HubSpot CRM (영업 도구 UX)

### 핵심 원칙
1. **한눈에 핵심 지표** — 로그인 즉시 오늘의 핵심 숫자가 보임
2. **정보 계층** — 요약 → 상세로 드릴다운
3. **빠른 액션** — 자주 쓰는 기능은 항상 접근 가능
4. **일관된 패턴** — 목록, 상세, 생성 페이지가 동일한 패턴

## 4. Design System

### 4.1 Color Palette

```
Primary:    #1E40AF (Blue 800)     — 주요 액션, 링크
Secondary:  #059669 (Emerald 600)  — 성공, 긍정적 지표
Warning:    #D97706 (Amber 600)    — 주의, 후속조치 필요
Danger:     #DC2626 (Red 600)      — 에러, 삭제, 위험
Neutral:    #374151 (Gray 700)     — 텍스트 기본

Background: #F9FAFB (Gray 50)      — 메인 배경
Surface:    #FFFFFF                 — 카드, 모달
Sidebar:    #111827 (Gray 900)     — 사이드바 배경 (다크)
```

### 4.2 Typography

```
제목 (H1):   text-2xl font-bold    (24px)
부제목 (H2): text-xl font-semibold  (20px)
섹션 (H3):   text-lg font-medium   (18px)
본문:        text-sm                (14px)  ← 대시보드 기본
보조 텍스트:  text-xs text-gray-500  (12px)
```

### 4.3 Spacing
- 카드 간격: `gap-4` (16px)
- 카드 내부 패딩: `p-6` (24px)
- 섹션 간격: `space-y-6` (24px)
- 인풋 간격: `space-y-4` (16px)

## 5. Responsive Strategy

| Breakpoint | 대응 |
|-----------|------|
| Desktop (≥1024px) | 사이드바 + 메인 콘텐츠 (기본) |
| Tablet (768-1023px) | 접이식 사이드바 |
| Mobile (< 768px) | 바텀 네비게이션 + 풀스크린 콘텐츠 |

> Phase 0-1은 Desktop 우선. 영업직원의 주 사용 환경이 PC.
> Phase 2부터 태블릿/모바일 대응 점진적 추가.

## 6. Loading & Empty States

### 로딩
- 페이지 전환: 상단 프로그레스 바 (NProgress 스타일)
- 데이터 로딩: Skeleton UI (shadcn/ui Skeleton)
- AI 생성: 스트리밍 텍스트 + 단계별 진행률

### 빈 상태
모든 빈 상태에 **다음 행동 가이드** 포함:
```
📄 아직 문서가 없습니다.

첫 번째 제안서를 AI로 만들어볼까요?
[+ 새 제안서 만들기]
```

## 7. Accessibility

- 키보드 네비게이션 지원 (Tab, Enter, Escape)
- 충분한 색상 대비 (WCAG AA 이상)
- 의미 있는 aria-label 사용
- 포커스 링 표시
- 스크린 리더 호환 (shadcn/ui 기본 제공)
