# New User Onboarding — Product Spec

## 1. Overview
코스콤 영업직원이 SalesHarness에 처음 접속했을 때의 경험을 정의합니다.
빈 화면이 아닌, 바로 가치를 체험할 수 있는 온보딩 흐름.

## 2. User Story
"As a 코스콤 영업직원, I want to 처음 로그인하면 바로 사용법을 알고 첫 제안서를 만들 수 있게, so that 도구 학습에 시간을 낭비하지 않는다."

## 3. Requirements

### Must Have (필수)
- [ ] 회원가입/로그인 (이메일 + 비밀번호)
- [ ] 프로필 설정 (이름, 부서, 담당 고객사)
- [ ] 첫 고객사 등록 가이드
- [ ] 첫 제안서 생성 체험 (샘플 데이터 활용)
- [ ] 주요 기능 투어 (3단계 이내)

### Nice to Have (선택)
- [ ] 샘플 데이터 프리로드 (데모 고객사, 데모 딜)
- [ ] 진행률 표시 (온보딩 3/5 완료)
- [ ] 건너뛰기 옵션

## 4. UI/UX Flow

```
로그인
  ↓
프로필 설정 (이름, 부서)
  ↓
"첫 고객사를 등록해볼까요?" [등록하기] [샘플로 시작]
  ↓
"AI로 제안서를 만들어볼까요?" [만들어보기]
  ↓
AI 제안서 생성 체험 (샘플 고객사 + PowerBase 상품)
  ↓
"준비 완료! 대시보드로 이동합니다." [시작하기]
```

### 빈 상태 처리
| 페이지 | 빈 상태 메시지 | CTA |
|--------|---------------|-----|
| 대시보드 | "환영합니다! 첫 고객사를 등록해보세요." | [고객사 등록] |
| 문서 | "아직 문서가 없습니다. AI로 첫 제안서를 만들어볼까요?" | [제안서 만들기] |
| 미팅 | "등록된 미팅이 없습니다. 다가오는 미팅을 추가해보세요." | [미팅 추가] |
| 파이프라인 | "진행 중인 딜이 없습니다. 첫 영업 기회를 등록해보세요." | [딜 추가] |

## 5. Data Model
- `users` 테이블: `onboarding_completed` (boolean), `onboarding_step` (integer)
- 프로필 완성도 체크: name, department 필수

## 6. API Endpoints
- `POST /api/auth/signup` — 회원가입
- `PUT /api/users/profile` — 프로필 업데이트
- `PUT /api/users/onboarding` — 온보딩 상태 업데이트

## 7. Edge Cases
- 이미 온보딩 완료한 사용자 → 대시보드로 바로 이동
- 온보딩 중 이탈 → 다음 로그인 시 이어서 진행
- 초대 없이 가입 시도 → "관리자에게 문의하세요" 안내

## 8. Success Criteria
- [ ] 가입 → 첫 제안서 생성까지 5분 이내
- [ ] 온보딩 완료율 80% 이상
- [ ] "건너뛰기" 사용률 30% 이하 (흐름이 자연스러운지 지표)
