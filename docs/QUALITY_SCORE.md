# SalesHarness — Quality Score

> 코드 품질, AI 출력 품질, UX 품질의 측정 기준과 목표를 정의합니다.
> "좋은 품질"이 무엇인지 합의하여 AI와 사람 모두 같은 기준으로 판단합니다.

## 1. Code Quality

### 정적 분석

| 도구 | 목적 | 규칙 |
|------|------|------|
| TypeScript | 타입 안전성 | `strict: true` |
| ESLint | 코드 스타일 | Next.js 기본 + 커스텀 규칙 |
| Prettier | 포맷팅 | 저장 시 자동 포맷 |

### 코드 메트릭 목표

| Metric | Phase 0-1 | Phase 2+ | 설명 |
|--------|-----------|----------|------|
| TypeScript strict | ✅ 필수 | ✅ 필수 | any 타입 금지 |
| 파일 크기 | < 400줄 권장 | < 400줄 필수 | 800줄 초과 시 분리 |
| 함수 크기 | < 50줄 | < 30줄 | 단일 책임 원칙 |
| 중첩 깊이 | < 4단계 | < 3단계 | 가독성 |
| 테스트 커버리지 | 핵심 로직만 | 80%+ | 점진적 강화 |
| 번들 크기 | 모니터링만 | < 300KB (초기 로드) | 성능 |

### 코드 리뷰 체크리스트

PR/커밋 시 확인:
- [ ] TypeScript 에러 0건
- [ ] ESLint 에러 0건 (warning은 허용)
- [ ] 네이밍 규칙 준수 (DESIGN.md 기준)
- [ ] 불변성 패턴 사용 (객체 직접 변경 없음)
- [ ] 에러 핸들링 적용
- [ ] 하드코딩 없음 (매직 넘버, 문자열)
- [ ] console.log 제거

## 2. AI Output Quality

### 문서 품질 기준

| 항목 | 기준 | 측정 방법 |
|------|------|-----------|
| 정확성 | 팩트 오류 0건 | 사용자 피드백 "오류 신고" |
| 완성도 | 수정 없이 사용 가능 비율 | 사용자 피드백 "그대로 사용" |
| 톤/스타일 | 비즈니스 한국어, 존댓말 | 샘플 리뷰 |
| 구조 | 섹션 누락 없음 | 템플릿 대비 체크 |
| 길이 | 적절한 분량 (너무 길거나 짧지 않음) | 사용자 피드백 |

### 품질 목표

| Metric | Phase 1 | Phase 2+ |
|--------|---------|----------|
| 사용자 "수정 후 사용" 비율 | 70%+ | 80%+ |
| 사용자 "그대로 사용" 비율 | 20%+ | 40%+ |
| 사용자 "사용 불가" 비율 | < 10% | < 5% |
| 평균 수정 시간 | 30분 이내 | 15분 이내 |

### 품질 개선 루프

```
사용자 행동 추적:
1. AI 생성 → 그대로 사용 (Good)
2. AI 생성 → 수정 후 사용 (OK, 수정 내용 학습)
3. AI 생성 → 폐기, 직접 작성 (Bad, 프롬프트 개선 필요)

피드백 저장:
- 각 생성물에 사용자 평가 (👍/✏️/👎) 수집
- 월별 품질 리포트 생성
```

## 3. UX Quality

### 핵심 지표

| Metric | Target | 측정 |
|--------|--------|------|
| LCP (Largest Contentful Paint) | < 2.5s | Vercel Analytics |
| FID (First Input Delay) | < 100ms | Vercel Analytics |
| CLS (Cumulative Layout Shift) | < 0.1 | Vercel Analytics |
| 3클릭 룰 | 핵심 기능 3클릭 이내 | 수동 검증 |
| 빈 상태 가이드 | 모든 빈 상태에 다음 행동 | 수동 검증 |

### UX 체크리스트

| 항목 | 기준 |
|------|------|
| 로딩 상태 | 모든 데이터 로딩에 Skeleton UI 표시 |
| 에러 상태 | 친화적 메시지 + 행동 가이드 |
| 성공 피드백 | 저장/생성 시 Toast 알림 |
| 반응형 | Desktop 기본, Tablet/Mobile 점진적 대응 |
| 접근성 | 키보드 네비게이션, 색상 대비 AA |
| 한국어 | UI 라벨 100% 한국어 |
