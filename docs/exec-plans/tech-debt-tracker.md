## Tech Debt Categories

### 🔴 Critical — 즉시 해결
- 보안 취약점
- 데이터 무결성 위험
- 프로덕션 장애 원인

### 🟡 High — 다음 Phase 전에 해결
- 성능 병목
- 중복 코드 (3회 이상)
- 테스트 없는 핵심 로직

### 🟢 Medium — 계획적으로 해결
- 코드 가독성 개선
- 리팩토링 기회
- 문서화 부족

### ⚪ Low — 시간 있을 때
- 더 나은 추상화 가능
- 의존성 업데이트
- 성능 미세 최적화

## Phase별 정리 계획

| 시점 | 작업 |
|------|------|
| Phase 0 → 1 | 코드 리뷰 기준 수립, ESLint 규칙 확정 |
| Phase 1 → 2 | 핵심 로직 테스트 추가, API 응답 표준화 |
| Phase 2 → 3 | DB 인덱싱, 컴포넌트 리팩토링, 번들 분석 |
| Phase 3 → 4 | 보안 감사, 성능 최적화, 코드 커버리지 80% |

## 등록 규칙

1. 발견 즉시 여기에 등록 (해결을 미루더라도 기록은 즉시)
2. 심각도 분류 필수
3. 해결 계획(어떤 Phase에서 해결할지) 명시
4. 해결 후 Resolved로 이동 + 해결 방법 기록
